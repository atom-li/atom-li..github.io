Eclipse - 导出JAR包

1、Jar包分为两种，一种是不可运行的，一种是可运行的Jar包，他们的主要区别如下：

  • 不可直接运行的Jar包主要是用于给别的程序提供调用
  • 可运行的Jar包主要是在运行程序时，可以直接用命令行编译运行，提高效率

2、先看一下不可运行的Jar包的制作过程：

  • 1、右键工程 -> Export… -> Java -> JAR file -> Next
  • 2、在Select the resources to export:下选择你要导出的工程
  • 3、下边有四个导出选项(一般选择第一个)：
      1. Export generated class files and resources : 导出类文件和资源文件
      1. Export all output folders for checked projects : 导出项目的所有生成文件
      1. Export Java source files and resoucrces : 导出Java源文件和资源文件
      1. Export refactorings for checked projects : 不知道干嘛用的
  • 4、 Select the export destination: 这里选择你到导出的源文件的名字和路径
    Options(选项)：
      1. Compress the contents of the JAR file : 是否要压缩JAR文件的内容
      1. Add directory entries : 是否选择添加目录项
      1. Overwrite existing files without warning : 是否可以未经警告改写现有文件
  • 5、接下来直接点 Finish，中间出现的所有弹窗，全选是，这样一个静态的Jar包就制作好了

3、可运行的Jar包的制作过程：

  • 1、右键工程 -> Export… -> Java -> Runnable JAR file -> Next
  • 2、在Launch configuration: 下选择你要导出的工程
  • 3、Export destination: 选择要导出的路径

  • 4、Library handing: 库的处理

    • Extract required libraries into generated JAR : 以解压的方式提取所需的库到生成的Jar包中
    • Package required libraries into generated JAR : 将需要的Jar包放到生成的Jar包中
    • Copy required libraries into a sub-floder next to the generated JAR : 将需要的库拷贝到一个子文件夹并放在生成的 jar 文件旁边

  • 5、接下来直接点 Finish，中间出现的所有弹窗，全选是，这里的Jar包就是可以单独运行的

4、Jar包的命令行用法：java -jar *.jar > server.log 2>&1
